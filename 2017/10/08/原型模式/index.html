原型模式

原型模式的原理

new一个对象，通过对象的clone方法实现对象复制就是原型模式
核心就是一个clone方法，通过Cloneable接口实现对象的拷贝

public class PrototypeClass {

@Override
public PrototypeClass clone(){
    PrototypeClass prototypeClass = null;

    try {
        prototypeClass = (PrototypeClass)prototypeClass.clone();
    }catch (Exception e){

    }
    return prototypeClass;
}

}

原型模式优点

性能优良
二进制流的拷贝，比直接new对象性能好
逃避构造函数的约束
拷贝对象不会执行构造函数

原型模式的使用场景

资源优化场景
需要消耗非常多的资源进行类初始化
性能和安全
new对象需要繁琐的数据准备和访问权限控制
一个对象多个修改者

注意事项

构造函数不会被执行：
Object类的clone方法的原理是从内存中以二进制流的方式进行拷贝，重新分配

内存块
浅拷贝和深拷贝
浅拷贝：只克隆基本类型和String类型的数据，不复制引用类型，对象中引用对象还是指向远对象
深拷贝：除浅拷贝外对应用对象也进行复制，注意循环引用问题
